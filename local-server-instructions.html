Running a Local Server for 3D Models

Why You're Seeing This Page

You're trying to view a website with 3D models (GLB files), but they're not loading correctly. This is likely due to CORS (Cross-Origin Resource Sharing) restrictions when opening HTML files directly from your file system.

The browser's security policy prevents loading local resources like 3D models when using the file:// protocol.

To properly view the 3D models, you need to run the website using a local web server. Here are several easy methods to do this:

Method 1: Using Python (Simplest)

If you have Python installed (most systems do), you can start a simple HTTP server with one command:

For Python 3:

python -m http.server

For Python 2:

python -m SimpleHTTPServer

After running this command:

  1. Open your browser and go to http://localhost:8000
  2. Navigate to the index.html file

Method 2: Using Node.js

If you have Node.js installed:

Install the 'serve' package (one-time setup):

npm install -g serve

Then run:

serve

Or without installing globally:

npx serve

After running this command:

  1. Open your browser and go to the URL shown in the terminal (usually http://localhost:3000)

Method 3: Using VS Code Live Server Extension

If you're using Visual Studio Code:

  1. Install the "Live Server" extension by Ritwick Dey
  2. Right-click on your index.html file
  3. Select "Open with Live Server"
  4. Your default browser will open automatically with the correct URL

Method 4: Using PHP (if installed)

php -S localhost:8000

Then open http://localhost:8000 in your browser.

Success Indicator

You'll know you're running the site correctly when:

  • The URL in your browser starts with http:// instead of file://
  • The 3D model appears in the hero section instead of the static fallback image
  • No CORS errors appear in the browser's developer console

Troubleshooting

If you still have issues:

  • Make sure your GLB file is correctly named hero-model.glb and placed in the assets/models/ directory
  • Check if the GLB file is valid and not corrupted
  • Check the browser console for specific error messages
  • Try a different browser
